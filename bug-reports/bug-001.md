## BUG-001 — Locked out user error message is not specific enough

| Field | Details |
|-------|---------|
| **Bug ID** | BUG-001 |
| **Feature** | Login |
| **Reported by** | Alison Kate |
| **Date Reported** | July 2026 |
| **Severity** | Minor |
| **Priority** | Low |
| **Status** | Open |
| **Environment** | Chrome, Windows, saucedemo.com |

---

### Description

When a locked out user attempts to log in, the error message displayed 
is generic and does not clearly tell the user that their account has 
been locked. This may confuse real users who do not know why they 
cannot log in.

---

### Steps to Reproduce

1. Open https://www.saucedemo.com
2. Enter username: `locked_out_user`
3. Enter password: `secret_sauce`
4. Click the Login button

---

### Expected Result

A clear error message should be displayed, such as:  
"Your account has been locked. Please contact support."

---

### Actual Result

The following error message is displayed:  
"Epic sadface: Sorry, this user has been locked out."

The phrase "Epic sadface" is unprofessional and does not clearly 
communicate the issue to the user.

## BUG-002 — Cart items persist after logout

| Field | Details |
|-------|---------|
| **Bug ID** | BUG-002 |
| **Feature** | Shopping Cart |
| **Reported by** | Alison Kate |
| **Date Reported** | July 2026 |
| **Severity** | Major |
| **Priority** | High |
| **Status** | Open |
| **Environment** | Chrome, Windows, saucedemo.com |

---

### Description

When a user adds items to the cart, logs out, and logs back in, 
the cart still contains the previously added items. Cart contents 
should reset after logout to prevent data leakage between sessions.

---

### Steps to Reproduce

1. Log in as `standard_user` with password `secret_sauce`
2. Add any product to the cart
3. Logout from the application
4. Log back in as `standard_user`
5. Click the cart icon

---

### Expected Result

Cart should be empty after logging back in. Items should not 
persist between sessions.

---

### Actual Result

Cart still contains the items that were added before logout. 
The cart badge shows the previous item count.
